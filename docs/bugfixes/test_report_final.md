## ❌ 发现的 Bug

### Bug 1: 已死亡玩家仍然有内心独白记录（严重）

**问题描述**: 
在游戏日志中，多个已死亡玩家仍然有 `inner_thought` 事件记录。

**影响范围**:
- 玩家 1、2、3、4、5、7、8 都有此问题
- 共检测到 20+ 次死人发言

**可能原因**:
在 `_run_discussion()` 方法中，代码在第 683 行获取了 `alive_players`，但在后续的发言循环中，如果有玩家在此期间死亡（例如猎人技能发动），仍然会记录他们的内心独白。

**代码位置**:
`core/game_engine.py` 第 709 行
```python
self._log_event("inner_thought", {
    "player_id": player.id,
    "thought": inner_thought,
    "round": round_num,
})
```

**修复建议**:
在记录内心独白前，检查玩家是否仍然存活：
```python
if player.is_alive:  # 添加此检查
    self._log_event("inner_thought", {
        "player_id": player.id,
        "thought": inner_thought,
        "round": round_num,
    })
```

---

### Bug 2: 预言家重复查验同一玩家（中等）

**问题描述**:
游戏日志显示预言家查验记录为 `[3, 8, 8, 9]`，玩家 8 被查验了 2 次。

**影响范围**:
- 浪费预言家查验机会
- 可能导致游戏平衡性问题

**可能原因**:
预言家 AI 的记忆系统可能没有正确记录已经查验过的玩家，或者 AI 决策逻辑有问题。

**代码位置**:
`ai/agent.py` 中的 `decide_night_action()` 方法

**修复建议**:
1. 在预言家行动前，传递已查验玩家列表
2. 在 AI 决策时排除已查验过的玩家
3. 添加验证逻辑，如果 AI 返回已查验玩家，则重新选择

---

### Bug 3: 警长竞选失败（轻微）

**问题描述**:
第一天警长竞选时出现 `no_votes` 情况，导致无人当选警长。

**影响范围**:
- 游戏缺少警长角色
- 可能影响游戏平衡性

**可能原因**:
- 部分 AI 投票逻辑可能弃权
- 参选玩家过少

**代码位置**:
`core/game_engine.py` 第 663 行 `_run_president_election()` 方法

**修复建议**:
1. 增加 AI 参选概率
2. 优化 AI 投票逻辑，减少弃权情况
3. 考虑添加重新竞选机制
