## 问题清单

### [致命] Bug #1: 预言家查验已死亡玩家

**问题描述**: 
预言家在夜晚行动时，查验了已经死亡的玩家（4 号）。日志显示：
- 第 3 夜：预言家查验了 4 号，但 4 号在第 2 天已被投票出局
- 第 4 夜：预言家再次查验了 4 号（已死亡）

**日志证据**:
```json
// 第 2 天投票结果
{
  "type": "player_eliminated",
  "data": {"player_id": 4, "role": "werewolf"}
}

// 第 3 夜预言家行动
{
  "type": "seer_check",
  "data": {"target": 4, "result": "werewolf"}  // 4 号已死亡！
}

// 第 4 夜预言家行动
{
  "type": "seer_check",
  "data": {"target": 4, "result": "werewolf"}  // 再次查验已死亡的 4 号！
}
```

**复现步骤**:
1. 运行游戏让预言家存活
2. 让某个玩家（如 4 号）在白天被投票出局
3. 观察后续夜晚预言家的查验目标

**预期行为**: 
预言家应该只能查验存活的玩家

**实际行为**: 
预言家查验了已死亡的玩家

**建议修复**: 
在 `_handle_seer_action()` 方法中，从可选目标中排除已死亡玩家：
```python
alive_others = [p.id for p in self.state.get_alive_players() if p.id != seer.id]
```

### [致命] Bug #2: 狼人袭击已死亡玩家

**问题描述**: 
狼人在夜晚行动时，选择了已经死亡的玩家作为袭击目标。

**日志证据**:
```json
// 第 3 夜狼人行动
{
  "type": "night_actions_summary",
  "data": {
    "wolf_action": {"target": 4}  // 4 号已在第 2 天死亡
  }
}

// 第 4 夜狼人行动
{
  "type": "night_actions_summary",
  "data": {
    "wolf_action": {"target": 7}  // 7 号已在第 1 夜死亡
  }
}
```

**复现步骤**:
1. 运行游戏让狼人存活多轮
2. 观察狼人夜晚袭击目标的选择

**预期行为**: 
狼人应该只能袭击存活的玩家

**实际行为**: 
狼人选择了已死亡的玩家作为目标

**建议修复**: 
在 `_handle_werewolf_action()` 方法中，确保从存活玩家中选择：
```python
alive_villagers = [p.id for p in self.state.get_alive_players() if p.role != Role.WEREWOLF]
```
并在 AI 决策时传递正确的可选目标列表。

### [严重] Bug #3: AI 内心独白固定不变

**问题描述**: 
所有 AI 玩家的内心独白都是固定的"我是狼人，要小心别暴露"，不管实际身份是什么。

**日志证据**:
```json
// 所有玩家的内心独白都一样
{
  "type": "inner_thought",
  "data": {
    "player_id": 1,  // 实际是女巫
    "thought": "我是狼人，要小心别暴露"  // 错误！
  }
}
{
  "type": "inner_thought",
  "data": {
    "player_id": 2,  // 实际是预言家
    "thought": "我是狼人，要小心别暴露"  // 错误！
  }
}
```

**原因分析**: 
DummyLLM 的 `generate_with_inner_thought` 方法中，判断条件有问题：
```python
if "狼人" in system_prompt:  # 这个条件总是匹配到系统提示中的"狼人杀游戏"
    speech = f"我是村民，昨晚没什么信息，先听大家发言。"
    inner_thought = "我是狼人，要小心别暴露"
```

**预期行为**: 
不同身份的 AI 应该有不同的内心独白

**实际行为**: 
所有 AI 都显示"我是狼人"的内心独白

**建议修复**: 
修改 DummyLLM 的判断逻辑，根据实际角色返回不同的内心独白：
```python
if player.role == Role.WEREWOLF:
    inner_thought = "我是狼人，要小心别暴露"
elif player.role == Role.SEER:
    inner_thought = "我是预言家，要保护好自己"
elif player.role == Role.WITCH:
    inner_thought = "我是女巫，手上有药"
```

### [严重] Bug #4: 猎人技能未测试

**问题描述**: 
在多次测试中，猎人从未有机会发动技能（被狼刀死亡）。

**日志证据**:
```json
"skill_usage": {
  "witch_heal": 0,
  "witch_poison": 0,
  "hunter_skill": 0
}
```

**原因分析**: 
测试中猎人要么存活到最后，要么被投票出局（不能开枪），从未被狼刀死亡。

**预期行为**: 
猎人被狼刀死亡时应该可以发动技能带走一人

**建议修复**: 
这不是代码 bug，但需要在测试中专门设计场景验证猎人技能：
1. 确保猎人是狼人袭击的目标
2. 验证猎人死后是否可以开枪
3. 验证被毒杀/投票时猎人不能开枪

### [一般] Bug #5: 女巫从未使用药剂

**问题描述**: 
在多次测试中，女巫从未使用过解药或毒药。

**日志证据**:
```json
"skill_usage": {
  "witch_heal": 0,
  "witch_poison": 0
}
```

**原因分析**: 
DummyLLM 返回的 action 是随机的，但女巫 AI 可能没有正确解析响应。

**预期行为**: 
女巫应该有机会使用解药自救，或使用毒药毒人

**建议修复**: 
检查 `_handle_witch_action()` 方法中 AI 响应的解析逻辑，确保正确识别 `heal` 和 `poison` 动作。

### [一般] Bug #6: 白天计数异常

**问题描述**: 
日志中 `day_number` 始终为 0，但游戏实际进行了多个白天。

**日志证据**:
```json
{
  "type": "day_start",
  "data": {
    "day": 0,  // 应该是 1, 2, 3...
    ...
  }
}
```

**原因分析**: 
在 `_run_day()` 方法中，`day_number` 在循环开始时递增，但日志记录时可能使用了错误的值。

**预期行为**: 
`day_number` 应该正确反映当前是第几天

**实际行为**: 
`day_number` 始终为 0

**建议修复**: 
检查 `game_engine.py` 中 `_run_day()` 方法的日志记录逻辑。
