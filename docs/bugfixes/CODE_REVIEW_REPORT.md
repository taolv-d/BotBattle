## 问题列表

### P0 - 致命问题

#### 问题 1: 女巫第一夜自救逻辑不完整

- **位置**: `core/game_engine.py:320-345`
- **问题描述**: 代码中虽然有第一夜自救的默认逻辑，但存在严重问题：
  1. 自救逻辑在 AI 返回无效 action 时才触发（第 329 行）
  2. 如果 AI 返回了有效但错误的 action（如毒杀自己），没有验证机制
  3. 自救条件判断不完整，没有检查女巫是否真的被刀
- **规范要求**: 
  > 参考 `docs/WITCH_ROLE_SPECIFICATION.md` 第 1.2 节：
  > "首夜被狼刀可以自救（BotBattle 规则）"
  > "第一夜被刀 → 必须自救"
- **现有代码**:
```python
# 第一夜且有人死亡（通常是狼刀目标），自动使用解药自救
if dead_player_id and not self.witch_heal_used and self.state.night_number == 1:
    action = {"action": "heal", "target": dead_player_id}
    self.witch_heal_used = True
```
- **建议修复**: 
  1. 在 AI 决策前明确告知女巫自己被刀
  2. 添加自救强制逻辑，而不是作为 fallback
  3. 验证毒药目标不能是自己
- **复现方法**: 
  1. 运行游戏，女巫在第一夜被狼刀
  2. 观察女巫 AI 是否正确自救
  3. 如果 AI 返回错误 action，观察自救是否触发

#### 问题 2: 预言家查验结果未正确传递给 AI

- **位置**: `ai/agent.py:358-362`
- **问题描述**: 
  1. 查验结果传递了，但格式不正确
  2. `seer_check_result` 是 Role 枚举，但 AI 需要的是"好人/狼人"二元结果
  3. 没有区分村民和神职，都显示为具体角色名
- **规范要求**:
  > 参考 `docs/SEER_ROLE_SPECIFICATION.md` 第 1.2 节：
  > "查验结果：仅返回'好人'或'狼人'，不区分具体身份"
  > "查验结果是二元的（好人/狼人），不区分具体身份"
- **现有代码**:
```python
# game_engine.py:419
self.state.seer_check_result = target.role  # 存储的是完整 Role 枚举

# agent.py:358
if is_seer and seer_check_target and seer_check_result:
    role_name = "好人" if seer_check_result.value == "villager" else "狼人"
    # Bug: 只判断了 villager，其他好人角色（女巫/猎人）会被误判为狼人
```
- **建议修复**:
```python
# 正确的判断逻辑
if is_seer and seer_check_target and seer_check_result:
    is_wolf = seer_check_result.value == "werewolf"
    role_name = "狼人" if is_wolf else "好人"
```
- **复现方法**:
  1. 运行游戏，预言家查验女巫或猎人
  2. 观察查验结果显示为什么
  3. 预期显示"好人"，实际可能显示"狼人"

#### 问题 3: 猎人技能发动条件判断错误

- **位置**: `core/game_engine.py:445-468`
- **问题描述**: 
  1. 猎人在夜晚被刀时立即发动技能，但此时猎人还未发表遗言
  2. 没有区分死亡原因（狼刀/投票/毒杀）就调用技能
  3. 被投票出局的猎人在 `_run_vote` 中明确不能开枪，但代码结构混乱
- **规范要求**:
  > 参考 `docs/HUNTER_ROLE_SPECIFICATION.md` 第 1.2 节：
  > "被狼刀死亡 → 可以开枪"
  > "被投票放逐 → 可以开枪"
  > "被女巫毒杀 → 不能开枪"
- **现有代码**:
```python
# game_engine.py:445-452
# 猎人技能 - 被狼刀死亡可以开枪
if target.role == Role.HUNTER:
    self.ui.display_system_message(f"{target.name} 是猎人，被狼刀死亡，可以发动技能！")
    self._handle_hunter_skill(target, alive_villagers)
```
- **建议修复**:
  1. 在 `_process_night_results` 中记录死亡原因
  2. 在遗言后、游戏结束前统一处理猎人技能
  3. 根据 `death_cause` 判断是否可以开枪
- **复现方法**:
  1. 运行游戏，猎人被女巫毒杀
  2. 观察猎人是否被允许开枪
  3. 预期不能开枪，实际可能可以开枪

#### 问题 4: 狼人袭击目标验证不完整

- **位置**: `core/game_engine.py:195-203`
- **问题描述**: 
  1. 验证了目标是否存活，但没有验证是否是狼人队友
  2. 如果 AI 返回了狼队友作为目标，代码会接受
  3. 只在注释中提到"不能袭击狼队友"，但无实际验证
- **规范要求**:
  > 参考 `docs/WEREWOLF_ROLE_SPECIFICATION.md` 第 2.2.1 节：
  > "不能袭击自己（除非自刀战术）"
  > "不能袭击狼队友"
  > "只能袭击存活非狼人玩家"
- **现有代码**:
```python
# game_engine.py:195
target = action.get("target")
if target is None or target not in alive_villagers:
    # 只验证了目标在 alive_villagers 中
    # 但 alive_villagers 定义是所有非狼人玩家
    # 如果 AI 返回狼队友，这里不会拦截
    import random
    target = random.choice(alive_villagers) if alive_villagers else None
```
- **建议修复**:
```python
# 获取所有狼人 ID
wolf_ids = [w.id for w in werewolves]
# 验证目标不能是狼队友
if target in wolf_ids:
    # 重新选择
    valid_targets = [p for p in alive_villagers if p not in wolf_ids]
    target = random.choice(valid_targets) if valid_targets else None
```
- **复现方法**:
  1. 运行游戏，观察狼人 AI 是否可能返回狼队友
  2. 如果有多个狼人，观察是否会互相袭击

#### 问题 5: 游戏结束条件判断过早

- **位置**: `core/game_engine.py:426`
- **问题描述**: 
  1. 在夜晚死亡处理后立即检查游戏结束
  2. 但猎人技能还未发动，可能改变局势
  2. 可能导致狼人提前获胜（猎人可以带走最后狼人）
- **规范要求**:
  > 参考 `docs/WEREWOLF_ROLE_SPECIFICATION.md` 第 1.3 节：
  > "狼人胜利条件：狼人全部死亡 → 好人胜利"
  > "好人胜利条件：所有狼人淘汰"
  > 
  > 参考 `docs/HUNTER_ROLE_SPECIFICATION.md` 第 1.2 节：
  > "猎人死亡时可以带走一名玩家"
- **现有代码**:
```python
# game_engine.py:426
# 应用死亡后
if killed_by_wolf and not saved:
    # ... 处理死亡
    # 猎人技能发动
    if target.role == Role.HUNTER:
        self._handle_hunter_skill(target, alive_villagers)

# 但在这之前就检查了游戏结束
self.state.check_game_over()  # 第 463 行
```
- **建议修复**:
  1. 在猎人技能发动后再检查游戏结束
  2. 确保所有死亡结算完成
- **复现方法**:
  1. 运行游戏至剩 1 狼 1 好人 1 猎人
  2. 狼人刀猎人
  3. 观察游戏是否在猎人开枪前结束
  4. 预期猎人开枪后判断胜负，实际可能提前结束
