## 4. 运行方式

### 4.1 初始化与配置

确保以下文件已准备好：

- `config/system.json`
- `config/werewolf_default.json`
- `config/personalities.json`

当前仓库的 `config/system.json` 已用于配置 LLM，运行时会从中读取模型、API Key、超时与重试参数。

### 4.2 启动游戏

```bash
python werewolf_main.py
```

启动后会依次询问：

1. 是否启用复盘
2. 复盘模式
3. 是否启用漏洞检测
4. 观察模式或玩家模式

## 5. 游戏流程

当前主流程由 `WerewolfOrchestrator.run_game()` 驱动：

1. 初始化游戏状态和角色 Agent
2. 进行警长竞选
3. 进入白天/夜晚循环
4. 满足胜负条件后结束游戏
5. 异步生成复盘报告

### 5.1 白天阶段

白天主要包括：

- 初始化发言顺序
- 玩家依次发言
- 狼人可在发言阶段自爆
- 若未自爆，则进入投票
- 若票出玩家，则发表遗言
- 若死者为警长，则处理警长继承

### 5.2 夜晚阶段

当前夜晚行动顺序为：

1. 守卫
2. 狼人
3. 女巫
4. 预言家
5. 结算死亡
6. 处理遗言与猎人技能

---

## 6. 关键规则

### 6.1 胜负条件

当前采用屠边规则：

- 狼人全灭：好人胜利
- 村民全灭：狼人胜利
- 神职全灭：狼人胜利
- 村民与神职同时全灭：狼人胜利

### 6.2 发言顺序

- 若警长存活，由警长选择左置位或右置位
- 若无警长或警长已死，则按玩家 ID 顺序发言

### 6.3 投票与平票

- 白天投票时，警长票权为 1.5 票
- 若第一轮平票，进入 PK 发言后再次投票
- 若再次平票，则无人出局

### 6.4 自爆

- 仅狼人可自爆
- 自爆后立即结束白天并进入夜晚
- 自爆玩家无遗言
- 若警长自爆，警徽流失

### 6.5 女巫信息隔离

当前实现遵循“深度信息隔离”：

- 女巫只知道“狼人是否刀人”
- 不知道被刀目标是谁
- 不知道守卫是否守中
- 即使刀的是自己，也不会直接获知目标身份

### 6.6 同守同救

当前规则为：

- 守卫和女巫同时作用于同一名普通玩家时，该玩家死亡
- 若女巫自救且守卫也守护女巫，则视为女巫自救成功，不算同守同救冲突

### 6.7 遗言

当前实现要点：

- 被票出：有遗言
- 自爆：无遗言
- 首夜死亡：有遗言
- 后续夜晚死亡：通常无遗言
- 同守同救死亡：有遗言

## 7. 复盘功能

狼人杀结束后会通过 `GameReviewService` 异步生成复盘结果。

### 7.1 支持内容

- 简要总结
- 详细报告
- 漏洞检测

### 7.2 输出位置

默认输出到：

- `reviews/review_<game_id>.md`
- `reviews/review_<game_id>.json`

### 7.3 当前实现说明

复盘服务已经集成到狼人杀主流程，但仍有一些可以继续优化的地方，例如：

- 更完整地利用真实日志事件
- 更稳健地解析漏洞检测结果
- 未来复用到三国杀

详见 [PENDING_ITEMS.md](PENDING_ITEMS.md)。
